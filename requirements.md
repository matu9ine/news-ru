# ТЗ — Новостной сайт «Цифровая редакция»

Простой новостной сайт.
Публикация от 3 новостей в день. Конструкторы сайтов запрещены.

---

## 1. Общие требования

- Публичная часть + админка.
- Стек: HTML / CSS / JS + Node.js для функциональной работы админки (через неё выкладываются новости).
- Дизайн — по `design.md` (минимализм, как у iz.ru: белый фон, красный акцент `#D32F2F`, строгий новостной стиль).
- Начального контента нет — новости добавляются через админку самим заказчиком (для проверки её работы).

---

## 2. Публичная часть

### 2.1. Главная страница
- Hero-новость (крупная): изображение 2/3, текст 1/3. Заголовок, краткое описание, дата, рубрика.
- Лента новостей — вертикальный поток. Каждая новость: заголовок, 2 строки описания, дата, рубрика. Hover: сдвиг вправо + красная полоса слева.
- Срочные новости — красная полоса-бегущая строка.
- Популярное — правая колонка, топ-5 заголовков.
- Карточки рубрик — последние 3 новости в каждой.

### 2.2. Страница рубрики
- Заголовок рубрики.
- Список новостей рубрики.

### 2.3. Страница новости
- Заголовок, дата, рубрика.
- Изображение.
- Текст: ширина 680–720 px, line-height 1.6–1.8.
- «Читайте также», кнопка «Назад».

### 2.4. Поиск
- Раскрывающийся input в шапке.
- Live-поиск (JS, без перезагрузки).
- Отдельная страница `/search`.

### 2.5. Адаптив
- Desktop: 3 колонки (контент + контент + сайдбар).
- Tablet: 2 колонки.
- Mobile: 1 колонка, бургер-меню, увеличенные кликабельные зоны. Hero: сначала текст, потом картинка.

### 2.6. Хедер
- Слева: логотип (текстовый / изображение).
- Центр: рубрики (Политика, Общество, Технологии, Экономика, Мир).
- Справа: иконка поиска, дата.
- Sticky, тонкая линия снизу, подсветка активной рубрики.

---

## 3. Админка

### 3.1. Новости
- Список (с фильтрами: статус, рубрика, поиск).
- CRUD.
- Визуальный редактор (WYSIWYG).
- Статус: черновик / опубликовано.
- Признак «срочная новость».
- Обложка (загрузка файла или URL).
- Slug (авто из заголовка или вручную).
- Дата публикации.

### 3.2. Рубрики
- CRUD.
- Поля: название, slug, порядок.

### 3.3. Администраторы
- CRUD (только для владельца).
- Роли: `owner`, `editor`.

### 3.4. Настройки сайта
- Название сайта.
- Логотип (текст + изображение).
- Соцсети: ВКонтакте, Telegram, Одноклассники, X.
- Настройки автопостинга.

---

## 4. Автопостинг

При переходе новости в статус «опубликовано»:

- **ВКонтакте** — `wall.post` через VK API.
- **Telegram** — `sendMessage` через Bot API.
- Пост: заголовок + ссылка на новость.
- Настройки (токены, идентификаторы) задаются в админке.
- Если настройки пустые — автопостинг пропускается без ошибки.

---

## 5. SEO

- `<title>` и `meta description` на всех страницах.
- OpenGraph: `og:title`, `og:description`, `og:image`, `og:site_name`, `og:type`.
- Twitter card.
- `schema.org` `NewsArticle` — JSON-LD на странице новости.
- Canonical URL.

---

## 6. Служебные файлы

- `favicon.svg` — минималистичный значок с буквой логотипа.
- `robots.txt` — `Allow: /`, запрет на `/admin` и `/api`, ссылка на sitemap.
- `sitemap.xml` — главная, рубрики, все опубликованные новости.

---

## 7. Ограничения

- Нельзя использовать конструкторы сайтов (Tilda, Wix и т. п.).
- Можно использовать Node-библиотеки (Express, Multer и т. д.).
- БД — встраиваемая, без отдельного сервера.
